# ADR-0005: 1ページ=1テーマ、レイアウトは写真枚数から自動決定

- Status: Accepted
- Date: 2026-07-18

## 状況

「Section(章)」概念を入れるか、レイアウトをユーザーに選ばせるかが論点だった。
締め切り 2〜3 日では、レイアウト選択 UI も選択作業自体もコストになる。

## 決定

- **1ページ=1テーマ**とし、Section 概念は導入しない。Album は Page の順序付き
  リストで完結する。
- **レイアウトはページ内の写真枚数から自動で決まる**。ユーザーは選ばない:
  - 1枚 → 大写真+キャプション(表紙・裏表紙もこの型)
  - 2枚 → 上下 2 段
  - 3〜4枚 → 2×2 グリッド(3枚なら 1 マス空きにキャプション)
- 1ページの写真は最大 4 枚。

## 帰結

- Layout はエンティティではなく `Page.photos.length` から導出される値(関数)。
- テンプレートは実質 3 種のみ。実装・テストが最小で済む。
- 「もっと凝せたい」は納品後の拡張とする(ADR-0001 の判断基準)。

## 改訂 (2026-07-18): コラージュモードと写真サイズ調整

グリッドでは 4 枚ページの写真が小さくなりがちという利用者の要望を受け、
自動導出の骨格は保ったまま 2 つの調整手段を追加した:

- **ページ単位のコラージュモード** (`Page.layout: "grid" | "collage"`、既定 grid)。
  写真枠を拡大(4枚・3枚は 1.3 倍、2枚は対角配置の専用枠)し、重なりを許して
  ポラロイド風の白フチで重ねる。描画は下の写真から行い、上段を前面に重ねる
  ことで各写真の下端(キャプション位置)が隠れないようにする。キャプションは
  白チップを敷いて最前面に描く。
- **写真単位のサイズ調整** (`Photo.scale`、0.7〜1.5 にクランプ、UI は小/標準/大/特大)。

レイアウト「選択」UI は引き続き作らない。枠の位置と枚数→レイアウトの対応は
自動のままで、利用者が調整するのは「大きさ」だけである。
