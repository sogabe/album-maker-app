# ドメインモデル — 自己紹介アルバムメーカー

決定の経緯は [ADR 0001〜0006](adr/)、用語の定義は [glossary.md](glossary.md) を参照。

## モデル

```
Album
 └─ pages: Page[]          # 順序付き。初期状態で8ページをプリセット(ADR-0006)
     ├─ title: string      # 英語ページタイトル(例 "My Family")
     └─ photos: Photo[]    # 1〜4枚(ADR-0005)
         ├─ file: string   # 写真フォルダ内のファイル名
         └─ caption: string # 短い英文(ADR-0003)

layoutOf(page): Layout      # 写真枚数から導出。1枚型 / 2段型 / 2×2グリッド型

PrintSpec(固定値): A4縦 / 300dpi / 余白5mm以上 / PDF 1ファイル・約10MB以内
```

エンティティは実質 Album・Page・Photo の 3 つ。Layout は導出値、PrintSpec は定数、
Section は存在しない。

## システム構成(ADR-0004)

```
写真フォルダ(AirDropで投入)
        │ 読み取り
        ▼
ローカル Web アプリ(Node 単一プロセス)
  - ブラウザUI: 写真の割り当て・並べ替え・キャプション編集・プレビュー
  - 永続化: album.json のみ
  - PDF生成: pdf-lib + sharp(300dpiリサイズ・サイズ上限管理)
        │
        ▼
album.pdf ──入稿──▶ コンビニ ネットワークプリント ──▶ 紙のアルバム
```

## スケジュール(締め切り: 2〜3日以内に印刷済み現物)

| 日 | 作業 | 担当 |
|----|------|------|
| Day 1 | 8テーマ分の写真をスマホから指名手配で選定・AirDrop(1〜2h)。並行してアプリ実装 | 家族 / Claude |
| Day 2 | アプリで割り当て・キャプション作成(息子と)→ PDF 出力 → 試し刷り1回 | 家族 |
| Day 3 | 修正 → 本番印刷 → 製本(クリアファイル等) | 家族 |

最大リスクは写真選定(特に「街」「学校」の撮り足し)。アプリ実装ではない。
